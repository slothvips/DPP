# 数据重建功能设计

**日期**: 2026-03-06

## 概述

在设置页面添加「数据重建」功能，允许用户清空本地数据并从服务器拉取完整数据。

## 需求背景

虽然同步机制已经完善，但难免出现数据不一致的情况。当用户发现本地数据与服务器不一致时，需要一个功能让用户可以重新从服务器拉取完整数据。

## 功能需求

1. **完全从远程重建**: 清空本地所有数据，从服务器拉取完整数据
2. **确认对话框**: 用户点击后需要二次确认，避免误操作
3. **自动同步**: 重建完成后立即触发同步

## UI 设计

### 位置
设置页面「数据管理」区域（导出/导入按钮旁边）

### 按钮样式
- 使用 Secondary 按钮风格
- 按钮文字：「重建本地数据」
- 添加警告图标提示风险

### 确认对话框
- 标题: 确认重建本地数据？
- 内容:
  ```
  此操作将清空所有本地数据并从服务器拉取最新数据。如果您有未同步到服务器的本地数据，这些数据将会丢失。

  ⚠️ 警告：为避免服务器压力，请仅在您的数据与至少两名团队成员不一致时使用此功能。
  ```

## 实现方案

### 核心逻辑

1. 用户点击「重建本地数据」按钮
2. 弹出确认对话框，说明风险
3. 用户确认后执行:
   - 调用 `syncEngine.clearAllData()` 清空本地所有数据
   - 调用 `syncEngine.pull()` 从服务器拉取完整数据
4. 显示成功/失败提示
5. 自动刷新页面

### 关键文件

| 文件 | 修改内容 |
|------|---------|
| `src/entrypoints/options/main.tsx` | 添加按钮和处理函数 |
| `src/lib/sync/SyncEngine.ts` | 确认方法可用 |

### 已有可复用方法

- `clearAllData()` (行 653-683): 清空所有本地数据和同步状态
- `pull()` (行 352-478): 从远程拉取操作记录

## 风险提示

- 此操作会丢失本地未同步的数据
- 需要确保用户已连接到服务器
- 重建后需要重新同步
