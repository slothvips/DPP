## 一、新增模块测试

### 1.1 热榜模块 (`lib/db/hotnews.ts`)

| 测试项       | 测试方法          | 预期结果                 | 验证方式（手动）         |
| ------------ | ----------------- | ------------------------ | ------------------------ |
| 获取热榜缓存 | 打开 Hot News Tab | 数据正常显示             | 观察列表是否正常渲染     |
| 保存热榜数据 | 刷新热榜          | 数据成功缓存到 IndexedDB | 再次打开时加载更快       |
| 清理旧数据   | 跨天后再次打开    | 只保留最近3天数据        | 观察数据条数是否符合预期 |

**手动验证要点：**

- 首次打开 Hot News Tab，等待数据加载完成
- 刷新页面，观察数据是否从缓存加载（速度更快）
- 切换不同日期，观察数据切换是否正确

## 五、数据一致性验证

### 5.1 软删除一致性

验证 Tags、Links、linkTags、jobTags 的软删除行为一致：

**手动验证方法：**

1. 创建一个测试 Link，然后删除它
2. 创建一个测试 Tag，然后删除它
3. 为 Link 关联 Tag，然后删除 Tag
4. 观察数据库中对应记录的 `deletedAt` 字段是否被设置

**验证工具：**

- 打开扩展的 Background 调试页面
- 查看 IndexedDB 中的数据，确认软删除字段

### 5.2 同步操作记录

验证活动记录功能正常：

**手动验证方法：**

1. 执行一些操作（如添加链接、触发构建等）
2. 在 AI Assistant 中询问 "最近几天做了什么操作"
3. 观察返回的操作记录是否包含刚才的操作
